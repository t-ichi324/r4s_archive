**R4S Archive System** は、セキュリティ、速度、そして物理構造の柔軟性が特徴の暗号化アーカイブソリューションです。

---

## 🚀 主な特徴

-   **🔑 可変長鍵 (8 / 16 / 32 バイト)**:
    -   `LOW (64-bit)`, `MID (128-bit)`, `HIGH (256-bit)` の 3 段階の鍵長をサポート。用途に合わせて、実用的な速度と究極のセキュリティを選択可能です。
-   **⚡ 高速 XOR 暗号化**:
    -   大容量データでも極めて高速に動作する XOR ストリーム暗号を採用。初期バイトへのスクランブル化により、既知のヘッダーパターン解析を防ぎます。
-   **🔐 強固なパスワードラッピング**:
    -   `SHAKE256` による動的な鍵導出と `SHA256` による整合性チェックを組み合わせ、パスワードの一致を厳密に管理します。
-   **🔄 セキュリティ・マイグレーション (Optimize)**:
    -   アーカイブ作成後でも、鍵長（セキュリティ強度）やパスワードを動的に変更・再構成可能です。
-   **📂 理論と物理の分離**:
    -   論理的なファイルパス管理と、バイナリレイアウトを完全に分離。メタデータ（UID/作成日時）の永続性を保証し、信頼性を高めています。

## 🛠 クイックスタート (CLI マネージャー)

`main.py` を使用して、すぐにアーカイブの操作が可能です。

### 1. アーカイブの作成
```bash
# 256-bit (HIGH) 鍵で新規作成
python main.py create my_secure_data.r4s -p "my_secret_pass" -k 32
```

### 2. ファイルの追加
```bash
python main.py add my_secure_data.r4s important_doc.pdf -p "my_secret_pass"
```

### 3. 一覧表示
```bash
python main.py list my_secure_data.r4s -p "my_secret_pass"
```

## 🏗 技術仕様

### 独自開発の4層構造 (4-Layer Structure)
R4S は、セキュリティと柔軟な運用の両立を目的とした、洗練された4層構造を採用しています：
1.  **第1層: 固定ヘッダー (Fixed Header)** - マジックナンバー、バージョン、UID、CRC1による整合性。
2.  **第2層: セキュリティ・レイヤー (Security Layer)** - KDFマスク、ラップされた物理鍵、パスワード検証用ハッシュ。
3.  **第3層: エントリー・インデックス (Manifest Index)** - 各エントリーの論理パス、オフセット、メタデータの管理。
4.  **第4層: 暗号化データ実体 (Encrypted Payload)** - XORストリームと先頭スクランブルによる暗号化データ。

### ヘッダー構造 (256 Bytes)
| Offset | Field | Description |
| :--- | :--- | :--- |
| 0 - 7 | Magic | `R4S_xx__` (xx は鍵長) |
| 10 - 11 | Key Length | 指定された鍵長 (8, 16, 32) |
| 64 - *** | Var Part | Mask, Scrambled Key, Integrity Hash |
| 252 - 255 | CRC32 | ヘッダー全体の整合性確認 |
